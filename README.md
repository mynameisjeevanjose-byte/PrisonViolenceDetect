ABSTRACT
The Smart Prison Management System (SPMS) integrates digital administrative management with AI-powered real-time security surveillance for correctional facilities. The system is built on a Flask-based Model-View-Controller (MVC) architecture supported by a SQLite relational
database managed via the SQLAlchemy ORM. A dedicated AI Surveillance Engine utilizes YOLOv8 (Ultralytics) for real-time weapon and contraband detection, and Google MediaPipe for skeletal pose estimation to automatically identify violent brawls (through Long Short-Term Memory
(LSTM) sequence modeling and wrist-velocity analysis) and inmate falls (through shoulder-ankle spatial heuristics).
The administrative modules cover the complete inmate life cycle: digital admission with autogenerated Nominal Roll IDs, risk-based automatic cell-block allocation, a Release Diary for tracking remand periods, medical health records management, daily attendance, and physical 
movement logging. Role-Based Access Control (RBAC) enforces strict data security across four staff roles: Superintendent, Jailer, Medical Officer, and Assistant Prison Officer (APO). All security threats detected by the AI engine are autonomously logged to an incident database 
and propagated to relevant dashboards in near real time. The system demonstrates how modern web technologies and computer vision can be effectively combined to modernize correctional facility operations, reduce administrative burden, and enhance proactive security monitoring.

Objective
The primary objectives of the Smart Prison Management System are:
1. Enhance Security: Integrate an AI-powered surveillance engine using YOLOv8 and MediaPipe to automatically detect weapons, violent brawls, inmate falls, and unauthorised gatherings in real-time from live CCTV feeds.
2. Centralise Data Management: Provide a unified relational database for managing the complete inmate lifecycle including admission (Nominal Roll), medical history, court dates, cell
allocation, daily attendance, and release dates.
3. Streamline Operations: Automate repetitive daily tasks for prison staff, including attendance marking, prisoner movement logging, and release date calculation, reducing administrative workload and error.
4. Ensure Accountability: Implement Role-Based Access Control (RBAC) so that sensitive
inmate data is accessible only to authorised personnel, and maintain automatic audit logs of
all security incidents.
5. Facilitate Timely Response: Propagate AI-generated security alerts to relevant staff dashboards in near-real time, ensuring rapid human intervention when automated threats are detected.

Implementation

The system is implemented as a full-stack web application using the Flask framework for the backend and standard HTML/CSS/JavaScript for the frontend. The core logic is written in Python,with SQLAlchemy serving as the Object-Relational Mapper (ORM) for database interactions. The
integration of computer vision and machine learning models provides autonomous surveillance
capabilities directly within the application’s backend architecture. The system’s functionality is delivered through several key Python functions and classes in the core application. Where applicable, specific algorithms are embedded directly within these functions
to handle complex data operations and predictive modeling.

Implementation of User Authentication & Routing

• login() and logout(): The login() function validates user credentials against the database,
establishes a secure session, and redirects the user to the appropriate role-based dashboard
(e.g., Superintendent, Jailor, Medical Officer). logout() safely clears the active session to
securely log the user out of the system.

Implementation of Inmate Admission and Management
• admit inmate() and edit inmate(): These functions handle the intake and modification
of prisoner records. They automatically generate unique nominal roll IDs, process mugshot
uploads, and save formal legal, contact, and medical details into the database.

Implementation of Real-Time AI Surveillance Engine
• CameraSystem.get frame(): This is the central engine for autonomous security monitoring. Running in a continuous background thread, it processes incoming video streams
frame-by-frame and sequentially applies multiple AI algorithms:
– Integrated Algorithm – YOLOv8 Object and Weapon Detection Algorithm: The
function utilizes a dual-model approach running the Ultralytics YOLOv8 algorithm. It
processes the resized image tensor using a pre-trained base model for standard objects
(like people and scissors) alongside a custom-trained model for contraband. It outputs
bounding box coordinates (x1,y1,x2,y2) and class IDs, highlighting detections that exceed confidence thresholds.
– Integrated Algorithm – Proximity and Crowding Spatial Algorithm: Operating on
the ’person’ bounding boxes extracted by YOLOv8, this spatial heuristic calculates 2D
centroids (cx,cy) for every pair of detected persons and computes the Euclidean distance
between them. If the distance falls below 120 pixels or bounding boxes intersect, a
”Proximity” flag is raised. Four or more proximate persons trigger a ”Crowding” escalation.
– Integrated Algorithm – MediaPipe Pose Extraction and Normalization: For behavioral analysis, the function estimates the skeletal pose of individuals, extracting 33
distinct 3D landmarks. A spatial normalization algorithm calculates the centroid of the
hips and shifts all landmarks relative to this center point, ensuring posture is analyzed
independently of the person’s screen location.
– Integrated Algorithm – LSTM Sequence Analysis for Violence Detection: The function maintains a continuous rolling window of the last 10 normalized pose frames. This
temporal sequence is fed into a custom bidirectional Long Short-Term Memory (LSTM)
network. The algorithm analyzes the sequential relationship of the joints over time and
outputs a Softmax probability across three classes (Normal, Brawl, Fall). If ’Brawl’
exceeds the confidence threshold, a violence alert is triggered.
– Integrated Algorithm – Heuristic Fall and Movement Algorithms: Acting as fallback triggers, deterministic spatial heuristics measure the vertical (Y) distance between
average shoulder and ankle heights for Fall Detection. Lunge and Kick detection calculate the frame-by-frame Euclidean velocity of wrists, ankles, and hips compared to
cached previous coordinates.
• log alert(): Called by the AI engine whenever an anomaly flags an anomaly, this function
autonomously logs the event type, location, and timestamp into the database while saving an
evidentiary snapshot of the incident.
Implementation of Dashboard and Routine Operations
• superintendent dashboard(): Aggregates facility-wide statistics, dynamically calculates
release countdowns to generate a diary, and enables the Superintendent to approve or reject
pending admissions.
• mark attendance() and log movement(): Core functions used by Assistant Prison Officers to update daily roll call statuses and record inmate movements between facility locations.
